# API Filtering and Pagination Guide for Frontend Developers

All the following endpoints use **GET** requests. Filtering and pagination parameters should be sent as **Query Parameters** (e.g., `?fieldName=value`).

---

## 1. Shared Pagination Parameters
Every search endpoint below supports standard Spring Data pagination:
- `page`: Page number (0-indexed, default: 0)
- `size`: Number of records per page (default: 20)
- `sort`: Sorting criteria in format `property,asc` or `property,desc` (e.g., `sort=centreName,asc`)

---

## 2. Endpoints and Filter Parameters

### A. Exam Applications
**Endpoint:** `GET /exam-applications`
**Available Filters:**
- `examId` (Long): Filter by Exam ID
- `studentId` (Long): Filter by Student ID
- `status` (String): Filter by Status (`SUBMITTED`, `APPROVED`, `REJECTED`)
- `regionId` (Long): Filter by Region ID
- `schoolId` (Long): Filter by School ID
- `examCentre` (Long): Filter by Exam Centre ID
- `applicationId` (Long): Filter by specific Application ID

**Example:** `/exam-applications?status=SUBMITTED&regionId=1&page=0&size=10`

---

### B. Exam Centres
**Endpoint:** `GET /exam-centres`
**Available Filters:**
- `centreName` (String): Partial match for Centre Name (case-insensitive)
- `centreCode` (String): Exact match for Centre Code
- `regionId` (Long): Exact match for Region ID

**Example:** `/exam-centres?centreName=Mumbai&regionId=2`

---

### C. Schools
**Endpoint:** `GET /schools`
**Available Filters:**
- `schoolName` (String): Partial match for School Name (case-insensitive)
- `examCentreId` (Long): Exact match for Exam Centre ID
- `regionId` (Long): Exact match for Region ID (searched via Exam Centre)

**Example:** `/schools?schoolName=High&examCentreId=5`

---

### D. Exam Results
**Endpoint:** `GET /exam-results`
**Available Filters:**
- `studentId` (Long): Filter by Student ID
- `examId` (Long): Filter by Exam ID
- `schoolId` (Long): Filter by School ID
- `regionId` (Long): Filter by Region ID
- `minPercentage` (Double): Results with percentage >= this value
- `maxPercentage` (Double): Results with percentage <= this value

**Example:** `/exam-results?examId=10&minPercentage=60.0`

---

### E. Students
**Endpoint:** `GET /students`
**Available Filters:**
- `firstName` (String): Partial match for First Name (case-insensitive)
- `lastName` (String): Partial match for Last Name (case-insensitive)
- `schoolId` (Long): Exact match for School ID
- `email` (String): Exact match for Student Email

**Example:** `/students?firstName=John&schoolId=3`

---

### F. Exams
**Endpoint:** `GET /exams`
**Available Filters:**
- `examName` (String): Partial match for Exam Name (case-insensitive)
- `examCode` (String): Exact match for Exam Code
- `status` (String): Filter by status (`DRAFT`, `PUBLISHED`, `CLOSED`, `RESULT_PUBLISHED`)

**Example:** `/exams?status=PUBLISHED&examName=Hindi`

---

### G. Regions
**Endpoint:** `GET /regions`
**Available Filters:**
- `regionName` (String): Partial match for Region Name (case-insensitive)

**Example:** `/regions?regionName=North`

---

## 3. How to send from Frontend (JavaScript/React)

Using `axios` or `fetch`, you should append these as query strings.

**Example with Axios:**
```javascript
const response = await axios.get('/exam-centres', {
    params: {
        centreName: 'Mumbai',
        regionId: 1,
        page: 0,
        size: 10,
        sort: 'centreName,asc'
    }
});
```

**Example with Fetch:**
```javascript
const query = new URLSearchParams({
    centreName: 'Mumbai',
    regionId: 1,
    page: 0,
    size: 10
}).toString();

const response = await fetch(`/exam-centres?${query}`);
const data = await response.json();
```
